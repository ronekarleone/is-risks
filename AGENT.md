# AGENT.md

## Назначение репозитория

Репозиторий используется для подготовки материалов курса по дисциплине «Управление рисками и процессами ИБ» для онлайн-платформы.

На текущий момент в репозитории ведется работа над модулем 1, уроком 1.

## Текущая структура

- `module-1/module-1-lesson-1-plan.md` — план урока для редакторов.
- `module-1/module-1-lesson-1-draft.md` — основной черновик урока.
- `Как оформить урок (мини-гайд д.pdf` — требования к оформлению уроков на платформе.
- `Паспорт_Управление рисками и процессами ИБ (2).xlsx` — паспорт дисциплины и структура курса.

## Текущий порядок работы

1. Сначала изучается паспорт дисциплины и требования по оформлению уроков.
2. Затем создается план урока в отдельном `md`-файле.
3. После утверждения логики создается черновик урока в отдельном `md`-файле.
4. Все содержательные правки вносятся в `draft`, а не в `plan`.
5. Если пользователь расставляет `TODO`, правки вносятся рядом с ними, но сами `TODO` не удаляются.
6. Если в уроке нужны изображения или схемы, в тексте оставляются явные промпты в квадратных скобках либо служебные маркеры вида `<picN>`.
7. Если изображения будут добавляться в проект, под них должна использоваться отдельная подпапка внутри соответствующего модуля.
8. После содержательных правок проверяется структура текста, логические переходы и соответствие роли урока в курсе.

## Правила редактирования черновика

- Урок 1 модуля 1 является вводным и не должен забирать глубину у модулей 2 и 3.
- Подробная идентификация и оценка рисков относятся к модулю 2.
- Подробная обработка рисков и меры защиты относятся к модулю 3.
- Общий подход по курсу: приоритет отдается не академическим теориям сами по себе, а практическим знаниям и способам мышления, которые используются в enterprise-среде.
- Теория включается только в том объеме, который помогает объяснить практику, управленческие решения, реальные роли, типовые сценарии и корпоративный контекст.
- В тексте нужно держать enterprise-фокус и управленческую перспективу.
- Визуальные элементы добавляются только там, где они реально упрощают понимание материала.
- Если абзац выбивается из основного хода изложения, предпочтительно выносить его в `Тогл`, `подложку` или отдельный служебный блок.
- Специальные блоки в драфтах нужно размечать явно не только началом, но и концом.
- Пример: `Тогл` ... `конец тогл`.
- Аналогично для других специальных блоков, если их границы могут быть неоднозначны при передаче редакторам.
- Весь материал должен подаваться как материал для студента. Запрещено делать текст в виде рекомендаций для преподавателя или редактора.
- Для всех терминов должны быть использованы их принятые в индустрии эквиваленты/аналоги/переводы на русском языке. В скобках должны быть указаны англоязычные оригиналы. Остаются на английском только Абберивиатуры и непереводимые термины. 
- Для вновь вводимых терминов в попапе должно быть приведено объяснение термина.
- Урок должен завершатсья списком литературы для самостоятального чтения. Для каждого источника должан быть указана ссылка, где можно эту литературу прочитать.

## Работа с изображениями

- В текущей сессии встроенный инструмент `image_gen` недоступен.
- Поэтому картинки готовятся внешне
- Для описания картинки нужно готовить детальный промт, описывающий изображение, который поймет редактор/художник без профильного образования

## Работа с git

- Репозиторий уже инициализирован.
- Основная ветка: `main`.
- Удаленный репозиторий: `origin = https://github.com/ronekarleone/is-risks.git`
- Перед коммитом нужно проверять, что изменения относятся только к текущей задаче.
- Если push не проходит, типовая причина — отсутствие аутентификации GitHub в среде.

## Практическое правило для следующих шагов

Если пользователь просит «снова пройтись по драфту», по умолчанию это означает:

1. найти новые `TODO`;
2. внести правки рядом с ними;
3. не удалять сами `TODO`, если пользователь явно не попросил обратное;
4. сохранить текущую структуру урока, если не запрошена переработка композиции целиком.
